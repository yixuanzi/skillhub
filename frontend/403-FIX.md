# 🔧 修复403 Forbidden错误指南

## 问题分析

从错误日志可以看到：
```
GET http://localhost:8000/api/v1/acl/resources/?page=1&pageSize=20 403 (Forbidden)
```

这说明请求**直接访问了 `localhost:8000`** 而不是通过Vite proxy。原因是：

1. ❌ Vite dev server使用了旧的环境变量配置
2. ❌ 浏览器可能缓存了旧的API配置

## ✅ 修复步骤

### 1. 停止Vite开发服务器

在运行 `npm run dev` 的终端中按 `Ctrl+C` 停止服务器。

### 2. 验证环境变量配置

确保 `frontend/.env` 文件内容为：

```bash
# API Configuration - Development
# Using relative path to leverage Vite's proxy to backend (localhost:8000)
VITE_API_BASE_URL=/api/v1
```

⚠️ **重要**：必须是相对路径 `/api/v1`，而不是完整的URL如 `http://localhost:8000/api/v1`

### 3. 清除浏览器缓存和localStorage

在浏览器开发者工具中执行：

```javascript
// 方法1: 清除所有localStorage
localStorage.clear()

// 方法2: 只清除认证相关的token
localStorage.removeItem('access_token')
localStorage.removeItem('refresh_token')

// 然后刷新页面
location.reload()
```

或者使用浏览器快捷键：
- **Mac**: `Cmd+Shift+R` (强制刷新)
- **Windows/Linux**: `Ctrl+Shift+R`

### 4. 重启Vite开发服务器

```bash
cd frontend
npm run dev
```

你应该看到类似的输出：
```
  VITE v5.x.x  ready in xxx ms

  ➜  Local:   http://localhost:5173/
  ➜  Network: use --host to expose
```

### 5. 重新登录

清除localStorage后，你需要重新登录：

1. 访问 `http://localhost:5173/login`
2. 输入用户名和密码
3. 登录成功后会重定向到dashboard

### 6. 访问Resources或ACL页面

现在应该可以正常访问了。如果一切正常，你应该看到：

- ✅ Network面板中请求URL是相对路径（如 `/api/v1/resources`）
- ✅ 请求通过Vite proxy转发到后端
- ✅ 返回200状态码而不是403

## 🔍 验证Proxy是否工作

打开浏览器开发者工具 (F12) → Network标签，查看API请求：

### ❌ 错误（直接访问后端）
```
Request URL: http://localhost:8000/api/v1/resources
Status: 403 Forbidden
```

### ✅ 正确（通过proxy）
```
Request URL: http://localhost:5173/api/v1/resources
Status: 200 OK
```

或者更简单的，在Network面板中查看"Remote Address"：
- ❌ 错误: `localhost:8000`
- ✅ 正确: `localhost:5173` (Vite dev server)

## 🐛 如果问题仍然存在

### 检查1: 确认后端服务器正在运行

```bash
# 检查8000端口是否被占用
lsof -i :8000

# 或者测试后端API
curl http://localhost:8000/health
```

### 检查2: 确认token存在

在浏览器Console中执行：

```javascript
console.log('Access Token:', localStorage.getItem('access_token'))
console.log('Refresh Token:', localStorage.getItem('refresh_token'))
```

应该看到两个token字符串。

### 检查3: 查看Vite proxy日志

在Vite dev server的终端中，你应该看到proxy日志：

```
Proxying: GET /api/v1/resources -> /api/v1/resources
```

如果没有看到这些日志，说明proxy没有工作。

### 检查4: 查看完整的错误信息

在浏览器Console和Network标签中查看：
- 请求的完整URL
- 请求头（是否包含 `Authorization: Bearer xxx`）
- 响应内容（后端返回的具体错误信息）

## 📝 相关文件

确保以下文件配置正确：

- ✅ `frontend/.env` - 环境变量配置
- ✅ `frontend/vite.config.ts` - Vite proxy配置
- ✅ `frontend/src/api/client.ts` - API客户端配置
- ✅ `backend/main.py` - 后端CORS配置
