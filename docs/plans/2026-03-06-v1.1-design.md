**Date:** 2026-03-06

## Executive Summary

SkillHub v1.1 introduces major enhancements to resource management, access control, and system observability. The release is split into two phases to reduce risk and enable incremental validation.

**Phase 1.1a** focuses on settings infrastructure including user-scoped API Keys and comprehensive audit logging.

**Phase 1.1b** introduces MCP (Model Context Protocol) resources, resource visibility controls, and admin-only user management.

### 1.1 API Keys

**Purpose:** Enable users to create personal, scoped API keys for programmatic access to SkillHub APIs.

#### Data Model

```python
class APIKey(Base):
    """User-scoped API key for authentication"""
    __tablename__ = "api_keys"

    id = Column(String(36), primary_key=True, default=lambda: str(uuid.uuid4()))
    user_id = Column(String(36), ForeignKey("users.id"), nullable=False, index=True)
    name = Column(String(255), nullable=False)  # User-defined name
    key_hash = Column(String(255), nullable=False, unique=True)  # SHA256 hash
    key_prefix = Column(String(10), nullable=False)  # First 10 chars for identification
    scopes = Column(JSONB, nullable=False, default=list)  # Permission scopes
    expires_at = Column(DateTime, nullable=True)  # Optional expiration
    last_used_at = Column(DateTime, nullable=True)  # Track usage
    is_active = Column(Boolean, default=True, nullable=False)
    created_at = Column(DateTime, default=lambda: datetime.now(timezone.utc))

    user = relationship("User", back_populates="api_keys")
```

**Key Format:** `sk_<32_random_characters>` (e.g., `sk_abc123xyz789...`)

#### Scopes

```python
class APIScope(str, enum.Enum):
    """API key permission scopes"""
    RESOURCES_READ = "resources:read"
    RESOURCES_WRITE = "resources:write"
    SKILLS_READ = "skills:read"
    SKILLS_CALL = "skills:call"
    SKILLS_WRITE = "skills:write"
    ACL_READ = "acl:read"
    ACL_WRITE = "acl:write"
    TOKENS_READ = "tokens:read"
    TOKENS_WRITE = "tokens:write"
```

#### API Endpoints

```
POST   /api/v1/api-keys/              Create API key (returns full key once)
GET    /api/v1/api-keys/              List user's keys (paginated)
GET    /api/v1/api-keys/{id}/         Get key details (no full key)
PUT    /api/v1/api-keys/{id}/         Update name/scopes/expires
DELETE /api/v1/api-keys/{id}/         Revoke key
POST   /api/v1/api-keys/{id}/rotate/  Generate new key, keep settings
```

#### Authentication Flow

```
1. Client creates API Key → Server generates "sk_xxx..." and returns it
2. Client stores key securely
3. Client includes key in header: "Authorization: Bearer sk_xxx..."
4. Middleware validates key hash and checks scopes
5. If valid, set user_id from key and proceed
6. Update last_used_at timestamp
7. Log action to audit_logs
```

#### Frontend Components

- **APIKeysPage**: List view with create/delete/revoke actions
- **APIKeyFormModal**: Create/Edit modal with name, scopes, expiration
- **APIKeyDisplayModal**: Show full key only on creation (never again)

```python
    },

}
```

#### MCP Service

```python
# services/mcp_service.py
class MCPService:
    async def call_mcp_resource(self, resource_id: str, method: str, params: dict):
        """Call an MCP server resource"""
        resource = Resource.get_by_id(resource_id)
        config = resource.ext

        if config["transport"] == "stdio":
            return await self._call_stdio(config, method, params)
        elif config["transport"] == "sse":
            return await self._call_sse(config, method, params)
        elif config["transport"] == "ws":
            return await self._call_ws(config, method, params)

    async def _call_stdio(self, config: dict, method: str, params: dict):
        """Execute MCP server via STDIO transport"""
        # Spawn subprocess, send JSON-RPC request, read response

    async def _call_sse(self, config: dict, method: str, params: dict):
        """Execute MCP server via Server-Sent Events transport"""
        # Connect to SSE endpoint, send request, wait for response

    async def _call_ws(self, config: dict, method: str, params: dict):
        """Execute MCP server via WebSocket transport"""
        # Connect to WebSocket, send JSON-RPC, wait for response
```

### 2.3 View Scope Implementation

**Purpose:** Control resource visibility independent of ACL permissions.

#### Visibility Rules

- **Public Resources:** Visible to all authenticated users in listings
- **Private Resources:** Only visible to owner and users with ACL permissions

#### ACL Integration

```python
class ACLResourceService:
    @staticmethod
    def list_all(db: Session, user_id: str, skip: int = 0, limit: int = 100):
        """List resources respecting view_scope and ACL"""

        # Public resources: always visible
        public_resources = db.query(Resource).filter(
            Resource.view_scope == "public"
        ).all()

        # Private resources: owner or ACL permission
        owned_resources = db.query(Resource).filter(
            Resource.view_scope == "private",
            Resource.owner_id == user_id
        ).all()

        # ACL-authorized private resources
        acl_resources = db.query(Resource).join(ACLRule).join(RoleBinding).join(UserRole).filter(
            Resource.view_scope == "private",
            UserRole.user_id == user_id
        ).all()

        return list(set(public_resources + owned_resources + acl_resources))

    @staticmethod
    def check_permission(db: Session, resource_id: str, check_data):
        """Check if user can access resource"""
        resource = Resource.get_by_id(resource_id)

        if resource.view_scope == "public":
            return PermissionCheckResponse(allowed=True, reason="Resource is public")

        if resource.view_scope == "private":
            if resource.owner_id == check_data.user_id:
                return PermissionCheckResponse(allowed=True, reason="User is owner")

            # Check ACL permissions
            acl_rule = ACLRule.get_by_resource_id(resource_id)
            if acl_rule and acl_rule.access_mode == "rbac":
                for binding in acl_rule.role_bindings:
                    if user_has_role(check_data.user_id, binding.role_id):
                        if check_data.required_permission in binding.permissions:
                            return PermissionCheckResponse(allowed=True)

            return PermissionCheckResponse(allowed=False, reason="Private resource - no permission")
```

## Security Considerations

### API Key Security

- **Key Storage:** Only key hash stored in database
- **Key Display:** Full key shown only once at creation
- **Key Rotation:** Users can rotate keys without changing permissions
- **Key Expiration:** Optional expiration date enforcement
- **Scope Limitation:** Keys only grant specified permissions
- **Rate Limiting:** Per-key rate limits (future enhancement)

### Audit Log Security

- **Immutable Records:** Audit logs cannot be modified
- **Admin Access:** Only admins can view audit logs
- **Sensitive Data Redaction:** Passwords, tokens automatically redacted
- **Retention Policy:** Automatic cleanup after configured days
- **Async Logging:** Non-blocking audit writes

### View Scope Security

- **Independent from ACL:** View scope is about visibility, not access
- **Private Resource Protection:** Private resources return 404 (not 403) to non-owners
- **Owner Verification:** Owner ID cannot be changed after creation
- **ACL Override:** ACL permissions override visibility for authorized users

### User Management Security

- **Admin-Only Access:** All user management endpoints require admin role
- **Audit Trail:** All user management actions logged
- **Role Validation:** Only existing roles can be assigned
- **Self-Protection:** Admins cannot remove their own admin role
